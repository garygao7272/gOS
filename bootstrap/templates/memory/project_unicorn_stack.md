**The Writ** is the name for God's command set — 9 slash commands that govern all creation. When Gary says "Writ", "Writ set", or "writ commands", he means this system.

**Soul file:** `.claude/GOD.md` defines God's identity, principles, voice, and relationship with Gary.

**Why:** User had 250+ surface-area tools (54 commands, 31 plugins, 18 agents) with no routing logic. Sessions started with "just talk" and the right tool was buried in a haystack. Based on LangChain article on coding agents reshaping EPD roles into Builders + Reviewers.

**The 9 Commands:**

| Command | Decree | Domain |
|---------|--------|--------|
| `/god` | Summon | Session entry, briefing, routing |
| `/think` | Reason | discover, design, research, decide, spec → specs/ |
| `/build` | Create | prototype, feature, component, test, deploy → apps/ |
| `/judge` | Evaluate | personas, specialists, full-council |
| `/schedule` | Automate | recurring agents on autopilot |
| `/coordinate` | Orchestrate | parallel sessions, handoffs |
| `/evolve` | Improve | self-reflect, upgrade commands, learn |
| `/prototype` | Sketch | quick prototype shortcut |
| `/verify-app` | Prove | verify it works |

**Key patterns:**
- Swarm for thinking (3-5 parallel agents), sequential for building (one commit at a time)
- Judge uses research-backed specialist personas with evidence-based verdicts (APPROVE/CONCERN/BLOCK)
- Evolve is the RL loop — audit, upgrade, learn, reflect, prompt-optimize
- Schedule uses Claude Desktop's built-in scheduled-tasks MCP
- mobile-design-engine is the best design tool in the ecosystem (user built it)

**Universal design:** The Writ is project-agnostic. GOD.md defines HOW we work. Each project's CLAUDE.md defines WHERE things live. To port the Writ to a new project, copy `.claude/commands/` + `.claude/GOD.md` — they adapt to whatever CLAUDE.md describes.
