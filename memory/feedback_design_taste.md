# Design Taste Preferences

Reference apps are the FLOOR, not the ceiling. AI+human co-creation surpasses what any single traditional team ships. Arx must beat every reference app, not match them.

**Why:** Gary corrected "reference ceiling" framing — these apps were built without AI co-creation. Arx has that advantage. The bar is ABOVE them, not AT them.

**How to apply:** When citing reference apps in design work, always frame as "minimum to beat" not "target to match." Never say "inspired by Robinhood" — say "surpassing Robinhood's onboarding by adding [specific Arx advantage]."

## Reference Floor (Persona-Split)

- **S7 (Followers, 95%):** Robinhood, eToro, Bitget, Phantom
- **S2 (Leaders, 5%):** Moomoo, Webull, Binance

## Key Corrections (2026-03-29)

1. **Taste belongs in specs/, not .claude/** — project-portable, part of artifact architecture (0-0), replicable across future projects. Created as `specs/Arx_4-3_Design_Taste.md`.

2. **Separate taste from design system** — 4-2 is tokens (material palette), 4-3 is judgment (the eye that picks the brush). Different audiences, change velocities, and altitudes.

3. **gOS weaknesses acknowledged:** Strong architect, weak designer. Specification masquerades as design. Scenario coverage is reactive. No reference research before specifying. Boldness deficit.

4. **Three new mandatory design gates:** Reference Research (before visual decisions), State Matrix (before spec writing), Feel Pass (after functional spec, before audit).
